1. ID тест-плана
   TP-SAMURAI-01
   
2. Цели тестирования
   Убедиться, что основные механики работают: движение, прыжок, атака, сбор монет, взаимодействие с врагами.
   Проверить, что игра не падает (не крашится) при переходе между уровнями.
   Подтвердить, что управление интуитивно понятно новому игроку.
   
3. Объём тестирования
   Входит в тестирование:
   Стартовый экран, меню выбора уровня
   Управление: клавиши (влево, вправо, прыжок, атака)
   Физика: движение по платформам, прыжки с края, гравитация
   Боевая система: атака, получение урона, смерть, возрождение
   Сбор бонусов (монеты, аптечки)
   Враги: передвижение, атака, исчезновение при убийстве
   Завершение уровня (собраны все монеты / достигнут выход)
   
   НЕ входит в тестирование (в первой версии):
   Звук и музыка
   Сохранение прогресса между игровыми сессиями
   Локализация (перевод на другие языки)
   Производительность на слабых ПК
   
4. Подход к тестированию
   Что проверяем	                                                               Техника / Тип
   Движение, прыжок, атака	                                                     Ручное функциональное тестирование
   Высота прыжка (минимум/максимум)	                                             Граничные значения (BVA)
   Комбинации (атака + прыжок, урон + смерть)	                                   Таблица решений
   Переходы между состояниями (стоит→бежит→прыгает→атакует)	                     Таблица переходов
   Сбор всех монет на уровне	                                                   Эквивалентное разбиение (0 монет, 1 монета, все монеты)
   Столкновение с врагом, бонусом, платформой	                                   Предугадывание ошибок

5. Критерии входа (когда начинаем тестирование)
   Минимально играбельный билд (build) собран и запускается
   Базовые элементы управления реализованы (ходьба, прыжок)
   Разработчик исправил критические баги, которые блокируют тестирование

6.  Критерии выхода (когда заканчиваем тестирование)
   Все тест-кейсы из критического пути выполнены
   Нет открытых багов с приоритетом «Критический» (игра крашится, невозможно пройти уровень)
   Игрок может пройти хотя бы один уровень от начала до конца

7. Артефакты (документы)
   Чек-лист (20–30 пунктов)
   Тест-кейсы (минимум 10 для критического пути)
   Баг-репорты (оформляются в GitHub Issues)

8. График (пример)
    День	             Активность
     ПН	               Составление тест-плана и чек-листа
     ВТ	               Прогон критического пути (тест-кейсы)
     СР	               Тестирование границ (прыжки, атака)
     ЧТ	               Исследовательское тестирование + оформление багов
